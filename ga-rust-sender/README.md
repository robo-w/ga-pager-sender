# ga-rust-sender

> [!WARNING]
> This Rust implementation was fully generated by `GPT-5.3-Codex` based on the Java implementation.
> The Java implementation is provided as-is, without in-depth testing.
> This implementation is provided as-is only with minimal smoke-testing.
> See the LICENSE file for details.

Rust CLI sender for the same TAP-like alpha payload format used by the Java implementation in this repository.

## Features

- Matches Java payload frame formatting:
  - `STX` + `7-digit RIC` + `A` + `baudrate marker` + `function marker` + `message` + `4-hex checksum` + `EOT`
- Sends payload over TCP.
- Waits for gateway response and validates:
  - NACK (`0x15`) handling
  - 4-byte checksum echo
  - ACK byte (`0x06`)

## Build

```bash
cargo build
```

## Run

```bash
ga-sender <server> <port> <target-ric> <baudrate> <function-code> <message>
```

Examples:

```bash
ga-sender 192.168.42.100 10300 12345 1200 1 "Test Message"
ga-sender pager-gateway.local 10300 12345 BPS_1200 CODE_1 Test Message
```

### Parameter values

- `baudrate`: `512`, `1200`, `2400` (or `BPS_512`, `BPS_1200`, `BPS_2400`)
- `function-code`: `0`, `1`, `2`, `3`, `4` (or `DEFAULT`, `CODE_1`, `CODE_2`, `CODE_3`, `CODE_4`)
- `target-ric`: integer range `8..=2097151`
